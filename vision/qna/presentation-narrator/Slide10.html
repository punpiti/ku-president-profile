Answer I Will Actually Give

ข้อความที่ผมจะนำเสนอในสไลด์นี้

  • เวลาที่ควรใช้: 0.9 นาที
  • สำหรับผม หน้านี้คือการบอกว่า KU จะสร้างผลกระทบมากขึ้นได้ ต้องเริ่มจากการทำให้ foundation cores ของเราแข็งแรงจริงก่อน
  • เพราะถ้าฐานไม่แข็ง frontier ก็จะกลายเป็นเพียงคำใหม่หรือกระแสใหม่ที่ไม่มีแรงส่งจริง แต่ถ้ามีแต่ฐานเดิมแล้วไม่เชื่อมเข้ากับโจทย์โลกใหม่ ศักยภาพนั้นก็จะไม่ถูกแปลงเป็น impact มากพอ
  • เพราะฉะนั้น logic ของหน้านี้คือ strong foundations x dynamic frontiers
  • คำว่า dynamic frontiers ในที่นี้ไม่ได้หมายความว่าเราจะตั้งหน่วยงานใหม่ไปเรื่อย ๆ และไม่ได้หมายความว่าเราจะเอา patch มาครอบ core แต่หมายถึงการเอาจุดแข็งเดิมของ KU ไปเชื่อมกับโจทย์ที่กำลังเร่งตัว เช่น AI และข้อมูล ความยั่งยืน อาหารและระบบชีวภาพ เศรษฐกิจดิจิทัล หรือ policy และ governance
  • ถ้าดูจากตัวอย่างของเกษตร จะเห็นชัดเลยว่าโลกจริงไม่ได้เป็นเกษตรแบบแยกส่วนอีกต่อไป แต่เชื่อมกับข้อมูล sensor AI เศรษฐศาสตร์ การตลาด กฎหมาย และทั้ง value chain ตั้งแต่ต้นน้ำถึงปลายน้ำ เพราะฉะนั้น frontier ไม่ใช่ของใหม่ที่ลอยอยู่ข้างบน แต่คือชั้นที่ทำให้ core เดิมเข้าไปสร้างคุณค่าในโลกใหม่ได้มากขึ้น
  • ในอีกชั้นหนึ่ง ภาควิชาที่เป็น core ของตัวเอง ก็อาจเป็น patch ให้ภาคอื่นได้เหมือนกัน ถ้าสาขาที่แข็งแรงสามารถเปิดแนวคิดใหม่ วิชาใหม่ หลักสูตรใหม่ หรือโจทย์วิจัยใหม่ให้กับอีกสาขาหนึ่งได้
  • โจทย์ของมหาวิทยาลัยจึงไม่ใช่แค่ทำให้แต่ละสาขาเก่งขึ้นลำพัง แต่ต้องทำให้สาขาเหล่านั้นเป็น patch provider ต่อกันได้อย่างมีประสิทธิภาพ
  • ตรงนี้สำคัญมาก เพราะทุกวันนี้เราทำ MOU กับคนข้างนอกได้ง่าย แต่ทำความร่วมมือข้ามภาควิชาข้างในให้เกิดของจริงกลับยากกว่า ถ้าไม่มี mechanism ที่ทำให้คนคุยกันรู้เรื่อง เห็นประโยชน์ร่วมกัน และเดินงานต่อได้จริง การบูรณาการก็จะเกิดแค่บนกระดาษ
  • ตัวอย่างที่ผมคิดว่าใช้ได้กับหน้านี้คือ AIEP เพราะมันไม่ใช่การเอา AI ไปแปะให้ดูทันสมัย แต่เป็นการเอา AI ไปยืนบน domain knowledge จริงของแต่ละสาขา เด็กยังต้องเรียนสาขาหลักของตัวเองให้แน่นก่อน แล้วค่อยต่อยอด AI จึงเป็นตัวอย่างชัดว่า strong foundations มาก่อน dynamic frontier
  • และ AIEP ก็ยังพิสูจน์อีกอย่างว่า ถ้ามีเจ้าภาพและคุยกันรู้เรื่อง core ของสาขาหนึ่งสามารถกลายเป็น patch ให้สาขาอื่นได้จริง จนเริ่มขยับเป็นวิชา หลักสูตร และเส้นทางการเรียนรู้ที่จับต้องได้
  • สุดท้าย หน้านี้จึงไม่ได้บอกให้ KU เลิกเป็นมหาวิทยาลัยตามสาขา แต่บอกว่าเราต้องใช้ความแข็งแรงในสาขาเป็นฐาน แล้วจัดพลังวิชาการใหม่ตามโจทย์โลก เพื่อให้เกิดวิชาใหม่ หลักสูตรใหม่ โครงการร่วม และงานวิจัยที่สร้างผลกระทบมากขึ้นจริงครับ

Suggest Note

อะไรที่ผมควรและไม่ควรทำ

  • ควรเริ่มจากแกนใหญ่ให้ชัดว่า ผลกระทบมากขึ้น ต้องมาจาก ฐานที่แข็งแรง ไม่ใช่จากการวิ่งตาม frontier อย่างเดียว
  • ควรย้ำว่า dynamic frontiers คือการเอาจุดแข็งเดิมไปเชื่อมกับโจทย์โลก ไม่ใช่การตั้งหน่วยใหม่ไปเรื่อย ๆ
  • ควรใช้ตัวอย่างอย่าง เกษตร x data/sensor/AI/value chain หรือ AIEP เพื่อทำให้ภาพ core x frontier เห็นของจริง ไม่ลอยเป็น diagram
  • ควรใช้ประเด็น patch provider เป็น logic หลังบ้านของการบูรณาการ แต่ไม่ควรอธิบายเชิงศัพท์หรือกลไกมากเกินไปจนเสียจังหวะของสไลด์
  • ควรทำให้คนฟังเห็นว่าเป้าหมายคือให้การเชื่อมกันออกมาเป็น วิชาใหม่ หลักสูตรใหม่ โครงการร่วม และโจทย์วิจัยใหม่ ไม่ใช่แค่ MOU หรือการประกาศความร่วมมือ
  • ไม่ควรทำให้สไลด์นี้ฟังเหมือนจะทิ้งฐานเดิม หรือเหมือนจะลดคุณค่าของความรู้ลึกในสาขา
  • ไม่ควรทำให้สไลด์นี้ไหลไปเป็นรายละเอียดเชิงกลไกมากเกินไป เพราะหน้าที่ของหน้านี้คืออธิบาย academic engine ในเชิงหลักการและภาพรวม
