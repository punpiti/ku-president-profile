Version To Actually Say

เวอร์ชันที่พูดจริงให้พอดีกับเวลาของสไลด์นี้

สไลด์หน้านี้ ผมพูดตรงไปตรงมาครับ

ความเชื่อมั่นของสถาบัน ต้องไม่ขึ้นกับว่าเราโชคดีได้คนแบบไหน

ถ้ายังต้องหวังว่าได้ “คนดี” แล้วทุกอย่างจะดี แปลว่าเรายังออกแบบระบบไม่ดีพอ

โจทย์ของ governance จึงไม่ใช่แค่ควบคุม แต่คือ ออกแบบให้การตัดสินใจ “สม่ำเสมอ โปร่งใส และตรวจสอบได้” แม้ตัวคนจะเปลี่ยน

ผมไม่ได้บอกว่าไม่ต้องมีดุลยพินิจ

แต่ดุลยพินิจต้องอยู่ใน กรอบที่ชัด ไม่ใช่ปล่อยให้ discretion กลายเป็นตัวแทนของระบบ

ปัญหาที่เห็นชัดในมหาวิทยาลัยคือ บทบาทไม่ชัด อำนาจไม่ชัด และความรับผิดชอบไม่ชัด

เราใช้คำว่า “กรรมการ” กว้างเกินไป ทั้งที่จริงมีทั้งคนที่ “ตัดสิน” คนที่ “แทน” และคนที่ “ให้คำปรึกษา”

ถ้าบทบาทไม่ชัด สุดท้าย ความรับผิดชอบจะพร่า และดุลยพินิจจะไหลเข้าแทนระบบ

สิ่งที่ผมจะทำ มี 4 เรื่องชัด

หนึ่ง — Standardized Governance กำหนดบทบาท สายการตัดสินใจ และ workflow ให้ชัด ลดความผันผวนตามตัวบุคคล

สอง — Reduced Discretion (where it shouldn’t be) ลดดุลยพินิจในเรื่องที่ควรเป็นระบบ เพื่อปิดช่องความเสี่ยงเชิงโครงสร้าง

สาม — Transparent & Secure Evaluation แยกระบบประเมินออกจากผู้ถูกประเมิน ข้อมูลปลอดภัย และตรวจสอบได้จริง

สี่ — Council Data Dashboard ให้สภากำกับจาก “ข้อมูลชุดเดียวกัน” เห็นภาพรวมและความเสี่ยง โดยไม่ต้องลงไปแทรกแซงรายกรณี

ถ้าระบบนี้เกิดขึ้นจริง

ความเป็นธรรม ความต่อเนื่อง และความไว้วางใจ จะไม่ต้องผูกกับตัวบุคคลอีกต่อไป

แต่จะถูก “ออกแบบไว้ในระบบ” ตั้งแต่ต้น

What I'll Use

ของที่หยิบไปใช้จริงบนหน้านี้ แยกบทพูดตั้งต้นและข้อควรระวัง

Answer I Will Actually Give

ข้อความที่ผมจะนำเสนอในสไลด์นี้

  • เวลาที่ควรใช้: 1.0 นาที
  • สำหรับผม หน้านี้พูดเรื่องตรงไปตรงมาครับ คือความเชื่อมั่นของสถาบันต้องไม่ขึ้นกับว่าเราโชคดีได้คนแบบไหนในรอบนั้น
  • ถ้ายังต้องหวังว่าได้คนดีแล้วทุกอย่างจะดีเอง แปลว่าเรายังออกแบบระบบไม่ดีพอ
  • เพราะฉะนั้นโจทย์ของ governance คือทำให้กติกา บทบาท ขั้นตอน และข้อมูลชัดพอ จนการตัดสินใจมีความสม่ำเสมอ โปร่งใส และตรวจสอบได้ แม้ตัวคนจะเปลี่ยน
  • แต่ผมไม่ได้หมายความว่าไม่ต้องมีดุลยพินิจเลยนะครับ ดุลยพินิจยังจำเป็นในบางเรื่อง เพียงแต่ต้องเป็น ดุลยพินิจภายใต้กรอบที่ชัด ไม่ใช่ปล่อยให้ discretion กลายเป็นตัวแทนของระบบทั้งหมด
  • จุดที่ผมคิดว่าสำคัญมากในมหาวิทยาลัยคือ บ้านเราใช้คำว่า กรรมการ กว้างเกินไป ทั้งที่จริงบางคนมาทำหน้าที่ตัดสิน บางคนเป็นผู้แทน บางคนมาให้คำปรึกษา ถ้าบทบาทไม่ชัด อำนาจหน้าที่ก็ไม่ชัด แล้วสุดท้ายความรับผิดชอบก็พร่าไปด้วย
  • พอความรับผิดชอบไม่ชัด ดุลยพินิจก็จะไหลเข้าไปแทนระบบ และคนที่อยู่ในระบบก็จะไม่แน่ใจว่าใครตัดสินใจ ในฐานะอะไร และใช้เกณฑ์อะไร
  • สิ่งที่ต้องทำจึงมีทั้ง standardized governance, transparent and secure evaluation, และ council data dashboard
  • standardized governance คือกำหนดบทบาท สายการตัดสินใจ และ workflow ให้ชัด ลดคอขวดและลดความผันผวนตามตัวบุคคล
  • transparent and secure evaluation คือทำให้กระบวนการประเมินแยกจากผู้ถูกประเมินจริง ข้อมูลปลอดภัยจริง และตรวจสอบได้จริง ไม่ใช่แค่บอกว่าปลอดภัย
  • ส่วน dashboard สำหรับสภามหาวิทยาลัย มีไว้เพื่อให้สภากำกับเชิงนโยบายจากข้อมูลเชิงระบบ เห็นความคืบหน้าและความเสี่ยงร่วมกัน โดยไม่ต้องลงไปแทรกแซงรายกรณี
  • ถ้าทำได้ ความเป็นธรรม ความต่อเนื่อง และความไว้วางใจต่อสถาบันก็จะไม่ต้องผูกกับอุปนิสัยของคนบางคนอีกต่อไป

Suggest Note

อะไรที่ผมควรและไม่ควรทำ

  • ควรเริ่มจากประโยคใหญ่ให้ชัดว่า ความเชื่อมั่นของสถาบันต้องออกแบบไว้ในระบบ แล้วค่อยลงรายละเอียดเรื่องกติกา บทบาท และข้อมูล
  • ควรใช้ประเด็น เอะอะก็เรียกกรรมการหมด เป็นตัวแปลปัญหาเชิง governance ให้คนฟังเข้าใจเร็ว แต่ไม่ควรลากยาวจนกลายเป็นการบ่นเรื่องศัพท์
  • ควรย้ำว่าเราต้อง ลดการใช้ดุลยพินิจโดยไม่จำเป็น ไม่ใช่ตัดดุลยพินิจทิ้งทั้งหมด เพราะ slide นี้พูดเรื่องกรอบที่ดี ไม่ใช่ระบบที่แข็งทื่อ
  • ควรทำให้ workflow, secure evaluation, และ council dashboard ฟังเป็นเครื่องมือของความเป็นธรรมและ accountability ไม่ใช่เครื่องมือควบคุมคน
  • ควรย้ำว่าสภามหาวิทยาลัยใช้ dashboard เพื่อ กำกับเชิงนโยบาย ไม่ใช่เพื่อลงไปตัดสินรายกรณี
  • ไม่ควรทำให้สไลด์นี้ฟังเป็น moral lecture ว่าปัญหาอยู่ที่คนไม่ดี เพราะสารหลักของหน้าคือ institutional design
  • ไม่ควรทำให้สไลด์นี้กลายเป็น tech pitch หรือพูดเรื่องระบบดิจิทัลมากเกินสารของ governance
