ภาพสไลด์จริงและข้อความที่แกะออกมาจากภาพ อยู่ในการ์ดเดียวกัน

P1 ผ่านแล้ว

Version To Actually Say

เวอร์ชันที่พูดจริงให้พอดีกับเวลาของสไลด์นี้

สไลด์หน้านี้ ผมอยากชัดเจนเรื่องหนึ่งครับ

ธรรมาภิบาลที่ดี ต้องตั้งอยู่บน “ข้อมูลชุดเดียวกัน” ที่ทุกคนเห็นร่วมกัน

ถ้าฝ่ายบริหารเห็นตัวเลขชุดหนึ่ง สภาเห็นอีกชุดหนึ่ง และส่วนงานเห็นอีกชุดหนึ่ง

ต่อให้เราพูดเรื่องความโปร่งใสแค่ไหน

การตัดสินใจจะกลับไปอยู่ที่ความรู้สึก อำนาจ หรือการตีความเฉพาะหน้า

เพราะฉะนั้น หน้านี้ไม่ใช่ IT project

แต่คือ decision infrastructure ของมหาวิทยาลัย

ผมมองเป็น 3 ชั้น

DATA เราต้องมีโครงสร้างข้อมูลกลาง ที่เชื่อมกันทั้งระบบ รู้ว่าใครเป็นเจ้าของข้อมูล และข้อมูลไหนใช้ตอบคำถามอะไร

DECISION ข้อมูลต้องไม่หยุดที่รายงาน แต่ต้องเข้าไปอยู่ใน การจัดสรรทรัพยากร และการตัดสินใจจริง

INTELLIGENCE ข้อมูลต้องช่วยให้เกิด “ความเป็นธรรม”

ไม่ใช่แค่ดูตัวเลข แต่ต้องเห็น ภาระงาน บริบท และข้อจำกัดของแต่ละส่วนงาน

ก่อนจะตัดสิน

อีกเรื่องที่สำคัญมากคือ

ความโปร่งใสที่ดี ต้องไม่ทำให้คนกลัวข้อมูล

ถ้าข้อมูลถูกใช้เพื่อ “จับผิด” ระบบจะไม่เกิด

แต่ถ้าข้อมูลถูกใช้เพื่อ “สนับสนุน”

ความไว้วางใจจะเกิดเอง

เช่น ถ้าเห็นว่าหน่วยงานใดมีภาระงานสูง สิ่งที่ตามมาควรเป็นการช่วย ไม่ใช่การตำหนิ

และถ้าระบบทำให้ กรอกครั้งเดียว ใช้ได้ทั้งระบบ

คนจะเริ่มเชื่อระบบเร็วขึ้น

ปัญหาของหลายองค์กร ไม่ใช่ไม่มีข้อมูล

แต่คือ คนทำข้อมูล คนใช้ข้อมูล และคนตัดสินใจ ไม่ได้อยู่ในระบบเดียวกัน

เพราะฉะนั้น data governance ที่ดี ต้องทำให้ทุกฝ่าย เชื่อมกันอยู่ใน “ภาษากลางเดียวกัน”

ถ้าทำได้

การตัดสินใจของมหาวิทยาลัย จะโปร่งใส รับผิดชอบได้ และเป็นธรรมมากขึ้น

และสำหรับผม data governance คือ priority ของการเปลี่ยนแปลงนี้ครับ

What I'll Use

ของที่หยิบไปใช้จริงบนหน้านี้ แยกบทพูดตั้งต้นและข้อควรระวัง

Answer I Will Actually Give

ข้อความที่ผมจะนำเสนอในสไลด์นี้

  • เวลาที่ควรใช้: 1.0 นาที
  • สำหรับผม หน้านี้คือการบอกว่า ธรรมาภิบาลที่ดีต้องตั้งอยู่บนข้อมูลที่คนในระบบมองเห็นร่วมกัน ไม่ใช่ต่างคนต่างมีตัวเลขของตัวเอง แล้วค่อยไปเถียงกันทีหลัง
  • ถ้าฝ่ายบริหารเห็นข้อมูลชุดหนึ่ง สภาเห็นอีกชุดหนึ่ง และส่วนงานเห็นอีกชุดหนึ่ง ต่อให้เราพูดเรื่องความโปร่งใสมากแค่ไหน สุดท้ายการตัดสินใจก็จะกลับไปอยู่บนความรู้สึก อำนาจ หรือการตีความเฉพาะหน้า
  • เพราะฉะนั้นหน้านี้จึงไม่ใช่เรื่อง IT project แต่เป็นเรื่อง decision infrastructure
  • กล่องแรกคือ DATA เราต้องมีโครงสร้างข้อมูลกลางที่เชื่อมถึงกันทั้งระบบ รู้ว่าใครเป็นเจ้าของข้อมูล ใครเป็นเจ้าของกระบวนการ และข้อมูลใดใช้ตอบคำถามแบบไหนได้บ้าง
  • กล่องที่สองคือ DECISION ข้อมูลที่ดีต้องไม่หยุดอยู่ที่การรายงาน แต่ต้องเข้าไปช่วยเรื่องการจัดสรรทรัพยากร การเห็นข้อจำกัด และการตัดสินใจบนฐานข้อมูลจริง
  • กล่องที่สามคือ INTELLIGENCE คือการวิเคราะห์เพื่อความเป็นธรรมของสถาบัน ไม่ใช่แค่ว่าใครมีตัวเลขมากหรือน้อย แต่ต้องช่วยให้เราเห็นบริบท ภาระงาน ข้อจำกัด และความแตกต่างของส่วนงานต่าง ๆ ก่อนจะตัดสินใจ
  • อีกเรื่องที่สำคัญมากคือ ความโปร่งใสที่ดีต้องไม่ทำให้คนรู้สึกว่าข้อมูลถูกใช้เพื่อจับผิดหรือทำให้อับอาย
  • ถ้าข้อมูลสะท้อนว่าหน่วยงานใดมีภาระงานสูง สิ่งที่ตามมาควรเป็นการสนับสนุน ไม่ใช่การตำหนิ และถ้าระบบช่วยให้คนกรอกครั้งเดียวใช้ได้หลายระบบ ทำงานง่ายขึ้น ความไว้วางใจก็จะเกิดเร็วขึ้น
  • สำหรับผม ปัญหาของหลายองค์กรไม่ใช่ไม่มีข้อมูล แต่คือผู้บริหารไม่เห็นข้อมูล ขณะที่คนทำข้อมูลก็ไม่รู้ว่าผู้บริหารต้องตัดสินใจภายใต้เงื่อนไขอะไร
  • เพราะฉะนั้น data governance ที่ดีต้องทำให้คนทำข้อมูล คนใช้ข้อมูล และคนรับผลของการตัดสินใจ เชื่อมกันอยู่ในภาษากลางเดียวกัน
  • ถ้าทำได้ การตัดสินใจของมหาวิทยาลัยก็จะโปร่งใส รับผิดชอบได้ และเป็นธรรมมากขึ้นครับ
  • data governance หรือการจัดการข้อมูล เป็น priority ของผม

Suggest Note

อะไรที่ผมควรและไม่ควรทำ

  • ควรเริ่มจากประโยคใหญ่ให้ชัดว่า ธรรมาภิบาลที่ดีต้องตั้งอยู่บนข้อมูลที่ทุกคนมองเห็นร่วมกัน
  • ควรย้ำว่า slide นี้พูดเรื่อง decision infrastructure ไม่ใช่ project IT หรือ dashboard เพื่อความสวยงาม
  • ควรใช้โครง DATA -> DECISION -> INTELLIGENCE เป็นแกนในการอธิบาย เพื่อไม่ให้สารกระโดดหรือกระจัดกระจาย
  • ควรทำให้คำว่า ความโปร่งใส ฟังเป็น โปร่งใสเพื่อการตัดสินใจที่เป็นธรรม ไม่ใช่โปร่งใสเพื่อจับผิดหรือทำให้อับอาย
  • ควรใช้ตัวอย่างอย่าง กรอกครั้งเดียวใช้ได้หลายระบบ, single source of truth, หรือ ข้อมูลใช้เพื่อสนับสนุน ไม่ใช่ตำหนิ เพื่อทำให้สไลด์นี้ดูจับต้องได้
  • ควรทำให้เห็นว่าปัญหาของหลายองค์กรไม่ใช่แค่ไม่มีข้อมูล แต่คือข้อมูลไม่เชื่อมกัน และคนทำข้อมูลกับคนตัดสินใจไม่ได้ใช้ภาษาร่วมกัน
  • ไม่ควรปล่อยให้สไลด์นี้ไหลไปเป็นเรื่องเทคนิค เช่น data lake, API, หรือ architecture มากเกินไป เพราะสารหลักคือ governance และ fairness
  • ไม่ควรทำให้สไลด์นี้ฟังเหมือนการรวมศูนย์อำนาจผ่านข้อมูล เพราะ framing ที่ถูกคือ shared visibility เพื่อความรับผิดชอบร่วมกัน
